# UI/UX 优化方案说明

## 目标
- 提升平台选择页的视觉层次、信息可读性和交互反馈一致性
- 强化键盘可达性与焦点可见性，满足无障碍基础要求
- 优化移动端/窄窗口布局表现，避免内容拥挤
- 保持现有功能逻辑不变，仅调整样式与布局结构

## 设计方向
- 深色科技风 + 青蓝主色，强调清晰层次与高对比
- 明确的区域分隔：标题区 / 选择区 / 输入区 / 操作区 / 底部提示
- 轻量化布局，减少多余容器，提升渲染效率

## 具体改动
### 1) 视觉层次与布局
- 标题区加高并使用更醒目的主色，强化品牌入口
- 选择区改为“卡片式”容器，边框与背景对比更清晰
- 输入区加独立分区与边框，降低干扰并突出可编辑区域
- 底部提示栏增加分隔线，增强区块结构感

### 2) 颜色、字体与间距
- 主色调整为高对比青色，辅助色为蓝色，强调色为橙色
- 文本层级区分：标题 > 副标题 > 说明文字
- 统一按钮尺寸与间距，提高视觉对齐度

### 3) 交互反馈
- 按钮新增 hover / focus / active / disabled 状态
- 列表焦点边框强化，提升键盘导航可见性
- 输入框 focus 状态与背景变化，降低误操作

### 4) 无障碍
- 强化焦点样式：列表、按钮、输入框均有明确高亮
- 保持高对比配色，提升低视力环境可读性
- 保留键盘导航逻辑（↑↓/Tab/Enter/Esc）不变

### 5) 响应式与自适应
- 针对窄窗口（<=80列）优化：按钮纵向堆叠、容器宽度放大
- 减少边距占用，避免内容被挤压

### 6) 性能优化
- 减少多余容器层级，降低布局复杂度
- 合并与精简平台选择页相关样式规则
