# Cold Shower

No-flattery adversarial review for ideas, requirements, technical plans, products, pricing, markets, and major decisions. The skill helps an agent find where a proposal breaks before implementation or commitment.

## When to use it

- challenge a feature request before writing code
- review a technical plan for hidden assumptions or overengineering
- pressure-test a product, pricing, or market-entry idea
- identify failure modes, missing evidence, and kill criteria
- ask the agent to act as a devil's advocate or to avoid emotional validation

## Workflow

1. restate the proposal in one sentence
2. classify the decision type
3. list the top hidden assumptions
4. attack the most fragile assumption first
5. name likely failure modes and missing evidence
6. propose the smallest falsification test
7. give kill, revise, and continue criteria

## Output shape

The default response contains:

- Cold read
- Hidden assumptions
- Most fragile point
- Failure modes
- Missing evidence
- Smallest falsification test
- Kill / revise / continue criteria

## Boundaries

The skill attacks the idea, not the person. It should be blunt and evidence-bound, not performatively cruel. For market or competitor claims that depend on current facts, the agent should follow the environment's research or web-access rules instead of inventing examples.
