# Session Wrap

Close out the current coding session with a compact summary of finished work, key decisions, validation, and next steps.

## Overview

Use this skill when the user wants:

- a current-session wrap-up
- a handoff note for the next session
- a summary before stopping work or switching context

It focuses on outcomes and evidence, not a chronological tool log.

## Workflow

1. Inspect the working tree with `git status`, `git diff --stat`, and recent commits.
2. Summarize completed work by workstream.
3. Call out important decisions and verification already performed.
4. Capture open risks, unfinished items, and suggested next actions.
5. Generate a handoff note if the user asks for one.

## When to Prefer Another Skill

- Use `project-daily-summary` for all of today's sessions across projects.
- Use `commit-daily-summary` for a pure commit-based report.
