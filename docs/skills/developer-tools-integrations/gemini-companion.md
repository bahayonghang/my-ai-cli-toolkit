# Gemini Companion

Use this skill when the user wants a **Gemini CLI companion workflow** with explicit stages and boundaries.

It is a good fit when Gemini should:

- review or analyze first
- provide a second pass before implementation
- continue a prior flow with explicit scope
- help define the next bounded step instead of acting as a one-shot wrapper

## Best for

- review-first Gemini workflows
- second-opinion analysis
- staged follow-up execution
- explicit continuation boundaries

## Notes

- Do not overpromise hidden provider-native persistence unless it truly exists.
- If the user expects Codex lifecycle commands, redirect to `codex-companion`.
- Keep the workflow explicit and verifiable.
