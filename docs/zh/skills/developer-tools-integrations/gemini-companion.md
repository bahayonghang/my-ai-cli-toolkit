# Gemini Companion

当用户希望使用 **Gemini CLI 的 companion 式工作流**，并要求阶段清晰、边界明确时，使用这个 skill。

它适合 Gemini 先做：

- review / 分析
- second opinion
- 明确下一步最小动作
- 在有限范围内继续之前的流程

## 最适合

- review-first 的 Gemini 工作流
- second-pass 风险分析
- 分阶段 follow-up 执行
- 明确边界的任务续接

## 说明

- 不要过度承诺隐藏的 provider 原生持久化能力，除非环境确实支持。
- 如果用户真正想要的是 Codex 风格生命周期命令，应改用 `codex-companion`。
- 流程应保持显式、可验证。
