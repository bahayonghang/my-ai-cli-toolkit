# planning-with-files

基于文件的复杂任务规划系统，支持自动会话恢复。

## 概述

Planning with Files 实现了 Manus 风格的基于文件的任务规划方法。它创建结构化的规划文档（`task_plan.md`、`findings.md`、`progress.md`）来维护上下文并跟踪长期任务的进度。

## 功能特性

- 📋 **结构化规划** - 创建 task_plan.md 进行逐步执行
- 🔍 **研究追踪** - 维护 findings.md 记录发现和洞察
- ✅ **进度监控** - 更新 progress.md 记录已完成步骤
- 🔄 **会话恢复** - `/clear` 后自动恢复上下文
- 🎯 **智能触发** - 对需要 >5 次工具调用的任务自动激活

## 适用场景

适合使用此技能的情况：
- 启动复杂的多步骤任务
- 进行研究项目
- 构建跨多个文件的功能
- 可能需要多个会话的任务
- 任何需要 >5 次工具调用的工作

## 使用方法

技能对复杂任务自动激活，或显式触发：

```
为这个功能实现制定计划
```

```
为构建认证系统创建任务计划
```

## 生成的文件

### task_plan.md

包含结构化计划：
- 任务概述
- 步骤分解
- 依赖关系
- 成功标准

### findings.md

追踪发现：
- 研究洞察
- 技术决策
- 阻碍和解决方案
- 参考链接

### progress.md

监控执行：
- 已完成步骤
- 当前状态
- 下一步行动
- 时间估算

## 工作流程

1. **初始规划** - 创建包含结构化步骤的 task_plan.md
2. **执行** - 逐步完成任务，更新 progress.md
3. **研究** - 在 findings.md 中记录发现
4. **恢复** - 如果会话被清除，自动恢复上下文

## 会话恢复

执行 `/clear` 后，技能自动：
1. 检测现有规划文件
2. 读取当前进度
3. 恢复任务上下文
4. 从上次检查点继续

## 最佳实践

- 让技能对复杂任务自动创建文件
- 开始执行前审阅 task_plan.md
- 发现新信息时更新 findings.md
- 使用 progress.md 追踪完成情况和下一步
