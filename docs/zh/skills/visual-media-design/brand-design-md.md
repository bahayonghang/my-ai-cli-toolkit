# Brand Design MD

当用户希望生成的 UI **贴近某个品牌的设计语言**，并且希望先从 `getdesign.md` 获取结构化设计规范，再据此产出代码时，使用这个 skill。

典型触发场景包括：

- “做成 Apple / Notion / Claude / Stripe 的感觉”
- “参考某个品牌的设计风格生成页面”
- “把两个品牌的视觉语言混搭一下”

## 它会做什么

1. 识别品牌并映射到正确 slug
2. 使用 `npx getdesign@latest add <slug>` 获取对应 `DESIGN.md`
3. 提取颜色、排版、间距、圆角、阴影等具体 token
4. 基于这些明确规则生成 UI 代码，而不是只做模糊风格模仿

## 最适合

- 品牌化 landing page 或页面区块
- 对齐某个产品视觉语言的 UI 探索
- 已知品牌设计系统的风格迁移
- 有明确混搭要求的多品牌设计方向

## 说明

- 当用户直接点名品牌或可识别产品风格时，这个 skill 最有价值。
- 它强调精确复用设计 token，而不是泛泛“像某某风格”。
- 如果品牌不在支持列表里，应明确说明并建议最接近的替代方案。
