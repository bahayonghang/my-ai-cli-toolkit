# Brand Design MD

当用户希望生成的 UI **贴近某个真实品牌、产品或媒体站点的设计语言**，并且希望先从 `getdesign.md` 获取结构化设计规范，再据此产出代码时，使用这个 skill。

典型触发场景：

- “做成 Apple / Notion / Claude / Stripe 的感觉”
- “参考某个品牌官网风格生成页面”
- “把现有 React / Vue 组件改成某品牌风格”
- “把两个明确品牌的视觉语言混搭一下”

## 它会做什么

1. 通过本地 helper 加上实时 `getdesign list` 目录解析品牌
2. 用带 `--out` 的固定输出路径获取对应 `DESIGN.md`
3. 提取颜色、排版、间距、圆角、阴影等具体 token
4. 基于这些规则生成 UI 代码，而不是只做模糊风格模仿
5. 区分“新建原型”和“改现有项目组件”两条路径

## 最适合

- 品牌化 landing page 或页面区块
- 已知品牌设计系统的风格迁移
- 有明确混搭要求的多品牌设计方向
- 在保持原技术栈前提下做现有组件的品牌化改造

## 说明

- 当用户直接点名品牌或可识别产品风格时，这个 skill 最有价值。
- 支持品牌集是实时目录，不是写死的静态表；截至 2026-04-14，上游目录返回 66 个品牌。
- 如果品牌不在支持列表里，应给出最接近的实时候选，而不是瞎猜。
- 对于泛化的“更现代一点”、logo、海报、slides、图表或图片生成，请优先考虑其它 skill。
