# Design Artifact HTML

当用户要的是 **HTML-first 的 web 前端产品原型**，而不是换主题的 deck、图表或图片时，使用这个 skill。

典型触发场景：

- “做一个可以直接在浏览器里打开的 clickable prototype”
- “把这个产品 brief 做成 HTML mockup”
- “做一个多屏 onboarding flow demo，用 HTML 展示”
- “给我一个像真实产品一样的一页式产品概念稿”

## 它会做什么

1. 先把请求归类为合适的原型模式，例如单屏概念稿、流程原型、交互演示或多方向探索
2. 在生成界面前读取用户提供的截图、代码、设计 token 或产品 brief
3. 默认产出一个独立 HTML 文件，配合内联 CSS 和轻量 JavaScript
4. 围绕真实的产品任务、状态和交互来构建，而不是堆砌泛化营销区块
5. 返回生成文件路径、选用的模式，以及关键假设

## 最适合

- 全新的产品界面原型
- 可点击的流程演示
- dashboard / workspace 概念稿
- onboarding、表单、设置等前端流程探索
- 在一个 HTML 产物里比较几种前端产品方向

## 说明

- 这个 skill 的首版刻意锁定为 **HTML-only**。
- 如果用户提到现有应用或代码库，skill 仍应默认产出独立 HTML 原型；只有在用户明确要求时才转向原框架内改造。
- 对于品牌风格迁移、主题替换、图表、海报或 slides，它都不应作为首选。
- 最终结果应该像“产品设计原型”，而不是一个泛化的“现代 SaaS 落地页”。
