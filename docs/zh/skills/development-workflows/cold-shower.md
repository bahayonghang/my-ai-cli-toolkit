# Cold Shower

用于想法、需求、技术方案、产品、定价、市场进入和重大决策的“无情绪价值”反方审查技能。它的目标是在实现或承诺之前，先找出方案最可能崩在哪里。

## 适用场景

- 写代码前挑战一个功能需求
- 审查技术方案里的隐含假设和过度工程
- 给产品、定价或市场进入想法泼冷水
- 找失败模式、缺失证据和停损条件
- 要求 agent 扮演魔鬼代言人、不要夸奖、不要情绪安慰

## 工作流程

1. 用一句话复述提案
2. 判断决策类型
3. 列出最重要的隐含假设
4. 优先攻击最脆弱的假设
5. 指出可能失败模式和缺失证据
6. 提出最小反证实验
7. 给出 kill / revise / continue 标准

## 输出结构

默认输出包含：

- Cold read / 冷读
- 隐含假设
- 最脆弱点
- 失败模式
- 缺失证据
- 最小反证实验
- Kill / revise / continue 标准

## 边界

这个技能攻击的是想法，不是人。语气可以直接，但必须受证据约束，不能为了“毒舌”而编造事实。涉及市场、竞品、法律、医疗、金融等需要当前事实的判断时，应按环境规则触发 research 或 web-access，而不是凭空举例。
