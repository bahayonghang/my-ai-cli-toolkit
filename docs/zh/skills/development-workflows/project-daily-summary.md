# Project Daily Summary

按项目汇总今天的工作，把会话证据、提交记录和未提交改动合并成一份简洁日报。

## 概览

适用于用户想要：

- 跨多个仓库的当日工作总结
- 按项目拆分的日报
- 同时包含 commits 与未提交改动的汇总

默认输出会先按项目分组，再在每个项目下整理主要工作主题。

## 工作流

1. 确定今天的本地日期。
2. 识别可用证据来源，例如 Codex 会话、当前对话上下文和 git 历史。
3. 对每个项目收集 `git log`、`git status` 和 `git diff --stat`。
4. 将相关活动合并为每个项目 1 到 5 个主要工作主题。
5. 输出时明确区分已完成、未提交与下一步建议。

## 何时改用其他技能

- 只需要基于 commit 的日报时，用 `commit-daily-summary`。
- 只需要当前会话总结时，用 `session-wrap`。
