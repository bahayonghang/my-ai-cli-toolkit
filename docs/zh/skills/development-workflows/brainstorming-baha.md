# Brainstorming Baha

中文优先的头脑风暴工作流，用于在实现前把粗略创意转化为已确认的设计方案。它还带有可选的浏览器视觉伴侣，可展示原型、图表和布局对比。

## 适用场景

- 创建功能、组件、项目或修改行为之前
- 需要先澄清目的、约束和成功标准
- 需要在编码前比较多个设计方向
- 视觉原型、架构图或并排布局比文字描述更容易决策

## 工作流程

1. 先查看当前项目上下文、文件、文档和最近提交
2. 如果预计有视觉决策，用独立消息提供视觉伴侣
3. 每次只问一个澄清问题，优先使用具体选项
4. 提出 2-3 种方案，说明权衡并给出推荐
5. 分节展示最终设计，获得批准后停止

## 视觉伴侣

视觉伴侣服务器会从会话目录中提供最新 HTML 屏幕，并把浏览器里的选择记录为事件。只有当“看到”比“读到”更清楚时才使用。

主要脚本：

- `scripts/start-server.sh`
- `scripts/stop-server.sh`
- `scripts/server.cjs`
- `scripts/helper.js`
- `scripts/frame-template.html`

## 关键约束

- 设计获批前不要写代码或开始实现。
- 澄清阶段每轮只问一个问题。
- 视觉伴侣提议必须与澄清问题分开发送。
- 优先设计职责清晰、边界明确的小单元。
- 设计获批即结束流程，除非用户明确要求继续实现。
