# learn: 概念解剖

把一个概念拆成八个切面，找到共同深层结构，最后压成一句顿悟。适合处理“一个概念”的深度理解，不适合普通白话改写。

## 适用场景

- 用户要“解释一下”“概念解剖”“learn concept”
- 需要从多个角度深入理解一个概念、术语或思想
- 想把一个概念整理成 org-mode 洞见笔记

## 工作流程

1. 先确认概念的通行定义、常见误解和核心词素。
2. 从历史、辩证、现象、语言、形式、存在、美感、元反思八个方向各切一刀。
3. 再用第一人称做一次内观，把多个切面收束到深层结构。
4. 压缩成公式、一句话和 ASCII 结构图。
5. 用模板生成 org-mode 笔记，并在需要时创建 `~/Documents/notes/`。

## 主要资源

- 八维分析框架
- `references/template.org`
- `~/Documents/notes/` 下的概念解剖笔记

## 关键约束

- 输出必须是纯 org-mode，不能混入 Markdown
- 只处理一个概念
- 结构图只能用基础 ASCII 字符
- 技术概念不要硬写成玄学散文
- 保存失败时要直接把完整 org 内容回到对话里
