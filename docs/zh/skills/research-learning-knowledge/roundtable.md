# roundtable: 圆桌求真

这个技能用于把一个议题放进主持人主导的多人物讨论里，让不同真实人物的思想路径正面碰撞，而不是只做普通总结。

## 适用场景

- 用户要求圆桌讨论、结构化辩论、多人物观点碰撞
- 用户想让几位思想家或公共人物围绕一个问题展开交锋
- 用户需要的是有推进感的多视角讨论，而不是平铺直叙的综述

## 核心流程

1. 先把请求收束成一个中心议题。
2. 选择 3-4 位真实人物，要求他们之间存在明确张力。
3. 由主持人开场，并提出统一定义问题。
4. 默认只跑一轮讨论，不自动扩成长篇连续剧。
5. 主持人提炼最深争议点，并画出 ASCII 结构图。
6. 用户决定继续、停止、深入当前裂缝，或引入新人物。
7. 如果环境允许，再把讨论导出成 org-mode 笔记。

## 输出约定

- 首次响应应包含：主持人开场、参会人物列表、首轮讨论、主持人综述、控制菜单
- 默认使用立场化转述，不伪造逐字引语
- 导出文件时必须保持纯 org-mode

## 关键约束

- 真实人物要谨慎选择，必须能解释其在场理由
- 不能伪造精确引语、著作信息或历史细节
- 结构图只能使用 ASCII
- 每一轮只深挖一条最核心的裂缝

## 说明

- 这个技能追求的是“求真式碰撞”，不是热闹的角色扮演。
- 如果用户不继续，主持人应在首轮后停住，而不是默认无限展开。
