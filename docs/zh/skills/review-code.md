# review-code

六维度代码审查技能，支持自信度过滤和多级报告。

## 概述

Review-Code 提供全面的多维度代码审查能力。它从六个维度分析代码，使用自信度过滤机制确保只报告高价值发现，避免噪音干扰。

## 六个审查维度

| 维度 | 关注点 | 示例 |
|------|--------|------|
| 🐛 **逻辑错误** | Bug、竞态条件、边界情况 | 空指针、死循环 |
| 🔒 **安全漏洞** | 注入、认证、数据泄露 | SQL 注入、XSS |
| ⚡ **性能问题** | N+1 查询、内存泄漏 | 循环内数据库查询 |
| 🏗️ **架构设计** | 模式、耦合、可维护性 | 违反 SOLID 原则 |
| 📖 **代码质量** | 可读性、命名、复杂度 | 过长函数、魔法数字 |
| ✅ **测试覆盖** | 测试充分性、边界测试 | 缺失单元测试 |

## 使用方法

### 审查 PR

```
/review-code PR #123
```

### 审查文件

```
审查 src/services/auth.ts 的代码
```

### 审查目录

```
全面审查 src/api/ 目录
```

## 自信度过滤

每个发现都有自信度评分 (0-100%)：

| 等级 | 自信度 | 显示行为 |
|------|--------|----------|
| 🔴 高 | >80% | 始终显示 |
| 🟡 中 | 50-80% | 默认显示 |
| ⚪ 低 | <50% | 默认隐藏 |

```
# 只显示高自信度问题
/review-code --confidence high

# 显示所有发现
/review-code --confidence all
```

## 报告格式

### 摘要视图

```markdown
## 审查摘要

- 🔴 严重问题: 2
- 🟡 中等问题: 5
- 🟢 建议改进: 8

### 关键发现
1. [安全] SQL 注入风险 (auth.ts:45)
2. [性能] N+1 查询 (users.ts:123)
```

### 详细视图

```markdown
## 详细分析

### [安全] SQL 注入风险
- 位置: `src/auth.ts:45`
- 自信度: 95%
- 描述: 用户输入未转义直接拼接 SQL
- 建议: 使用参数化查询
- 代码示例: ...
```

## 最佳实践

1. **聚焦关键问题** - 优先处理高自信度发现
2. **上下文审查** - 结合业务场景评估严重性
3. **增量改进** - 不必一次修复所有问题
4. **团队标准** - 建立一致的代码质量标准
