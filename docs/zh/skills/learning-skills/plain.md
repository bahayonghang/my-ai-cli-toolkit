# plain: 白话改写

把复杂内容改写成聪明 12 岁也能复述的版本。它不规定形式，只卡低线：口语、具体、少术语、少废话。

## 适用场景

- 用户说“白话说”“说人话”“plain”“grok”
- 需要把技术、概念或说明改写得更易懂
- 想输出一篇顺畅、自然、无结构标签的 org 短文

## 工作流程

1. 先拿到内容来源：文本、链接、文件或概念。
2. 选择最适合主题的表达方式，比如类比、故事、问答或递进例子。
3. 逐条过红线，删掉拐杖词、空话、术语堆叠和不自然表达。
4. 用模板生成一篇从头流到尾的连贯正文，并在需要时创建 `~/Documents/notes/`。

## 主要资源

- 类比、故事、问题拆解和画面感这些写作工具
- `references/template.org`
- `~/Documents/notes/` 输出目录

## 关键约束

- 读起来必须像自然口语
- 术语要先落地解释，再出现术语名
- 正文里不要加结构标签或写作过程评论
- 保存失败时要直接把完整 org 内容回到对话里
- 它做的是白话改写，不是概念解剖或单词学习
