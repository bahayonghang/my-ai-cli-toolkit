# rank: 降秩

把一个领域砍到最少的独立生成器，找出真正撑起全部现象的那几根线。目标不是总结，而是还原。

## 适用场景

- 用户问“秩是什么”“这个领域靠什么撑着”
- 需要把一堆现象压缩成最小生成集合
- 想写成一篇有落差的短文，而不是表格

## 工作流程

1. 先收集或补齐足够代表性的现象。
2. 找出候选生成器，判断哪些因素真的独立。
3. 用四个判据检查：生成性、最小性、独立性、预测力。
4. 把验证过程融进散文式叙述里，而不是另起附录。
5. 最后明确给出“秩：N”或“秩：未定”，再写一句 takeaway。

## 主要资源

- 四个秩判据
- `references/template.org`
- `~/Documents/notes/` 下的输出文件

## 关键约束

- 必须能从生成器反向生成全部现象
- 任何冗余生成器都不算通过
- 叙述要像散文，不要写成条目堆砌
- 验证必须写进正文
- 若四个判据无法同时成立，就写“秩：未定”
