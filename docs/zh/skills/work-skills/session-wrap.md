# Session Wrap

为当前编码会话做收尾，总结已完成事项、关键决策、验证情况以及下一步建议。

## 概览

适用于用户想要：

- 当前会话总结
- 下一次继续工作的交接说明
- 在停止工作或切换上下文前做一次收尾

它关注结果和证据，而不是按时间顺序复述所有操作。

## 工作流

1. 用 `git status`、`git diff --stat` 和最近提交检查工作区状态。
2. 按工作主题总结本次会话的完成事项。
3. 记录关键决策和已完成的验证。
4. 提炼未完成项、风险和建议的下一步。
5. 如果用户需要，再生成结构化 handoff note。

## 何时改用其他技能

- 需要总结今天所有项目或所有会话时，用 `project-daily-summary`。
- 只需要基于提交生成日报时，用 `commit-daily-summary`。
