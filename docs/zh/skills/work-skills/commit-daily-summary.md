# Commit Daily Summary

把一天内的 git 提交整理成按工作主题分组的日报，而不是简单罗列提交时间线。

## 概览

适用于用户想要：

- 基于提交记录生成当日日报
- 快速回答“我今天做了什么”
- 从 commit 中提炼可读的工作总结

默认范围是当前仓库和用户本地时区下的“今天”。

## 工作流

1. 先确定日期范围，使用用户本地时区。
2. 通过 `git log --since ... --until ...` 收集提交。
3. 把相关提交合并成少量工作主题。
4. 将原始提交标题改写为可读的动作型摘要。
5. 如果用户需要可追溯性，再附上原始提交列表。

## 何时改用其他技能

- 需要跨多个仓库或结合会话证据时，用 `project-daily-summary`。
- 只想总结当前会话时，用 `session-wrap`。
