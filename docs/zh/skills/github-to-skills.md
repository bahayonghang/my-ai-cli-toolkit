# github-to-skills

将 GitHub 仓库转换为 AI 技能的自动化工厂。

## 概述

GitHub-to-Skills 是一个自动化工厂，可以将 GitHub 仓库转换为可用的 AI 技能。它分析仓库结构、文档和代码，生成符合标准的技能定义。

## 功能特性

- 🏭 **自动化工厂** - 批量处理多个仓库
- 📊 **智能分析** - 自动识别仓库类型和功能
- 📝 **文档提取** - 从 README 和文档中提取技能描述
- 🔧 **配置生成** - 创建推荐配置
- 📋 **模式识别** - 识别常见使用模式

## 使用方法

### 单个仓库

```
将 https://github.com/user/repo 转换为技能
```

### 批量处理

```
从这个 awesome 列表创建技能：https://github.com/user/awesome-xxx
```

## 转换流程

1. **仓库分析**
   - 克隆或获取仓库元数据
   - 分析项目结构
   - 识别主要功能

2. **文档提取**
   - 解析 README.md
   - 提取 API 文档
   - 收集示例代码

3. **技能生成**
   - 创建 SKILL.md
   - 生成配置模板
   - 添加使用示例

4. **质量检查**
   - 验证生成内容
   - 确保格式一致性

## 支持的仓库类型

| 类型 | 描述 | 输出 |
|------|------|------|
| CLI 工具 | 命令行工具 | 命令使用指南 |
| 库 | 代码库/SDK | API 使用技能 |
| 框架 | 开发框架 | 开发指导技能 |
| 文档 | 文档仓库 | 参考技能 |

## 输出结构

```
skills/[repo-name]/
├── SKILL.md          # 主技能定义
├── config/           # 配置模板
├── tips/             # 使用技巧
├── references/       # 原始文档链接
└── cookbook/         # 代码示例
```

## 最佳实践

1. **选择活跃仓库** - 优先选择维护良好的项目
2. **验证许可证** - 确保兼容的开源许可
3. **检查文档** - 选择文档完善的仓库
4. **测试输出** - 验证生成的技能是否可用
