# 论文流 (Paper Interpretation Flow)

一条命令串起“读论文 + 做卡片”：先生成论文解读，再把解读铸成视觉卡片，适合多篇论文并行处理。

## 适用场景

- 用户一次给出多篇 arXiv 链接、PDF 或论文名
- 需要同时产出论文解读和配套卡片
- 想把阅读结果沉淀成可分享的视觉产物

## 工作流程

1. 收集用户消息里的全部论文来源。
2. 对每篇论文先执行 `paper-interpretation`，生成 org 解读文件。
3. 再读取该 org 文件，调用 `card` 生成对应 PNG 卡片。
4. 所有论文完成后，汇总返回每篇的解读路径和卡片路径。

## 主要资源

- `paper-interpretation`：论文解读主技能
- `card`：把解读内容渲染成 PNG
- 论文来源可以是链接、PDF、文件路径或论文名

## 关键约束

- 每篇论文内部必须先解读、后制卡，顺序不能倒
- 多篇论文可以并行，但单篇的两步必须串行
- 卡片内容来源于解读结果，不直接吃原始论文
