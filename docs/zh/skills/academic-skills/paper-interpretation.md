# 读论文 (Paper Interpretation)

面向非学术读者的读论文技能，把论文转成可直接吸收和使用的理解，而不是学术式批评。

## 适用场景

- 用户提供 arxiv 链接、论文 URL、PDF 或本地论文文件
- 用户希望“读论文”“分析论文”或理解某篇研究工作
- 目标是提炼可用思想，而不是产出正式学术评审

## 核心流程

1. 从 URL、PDF、本地文件或论文标题收集论文内容
2. 找到论文真正试图解决的问题
3. 用面向聪明外行的白话解释核心方法
4. 提炼一到三个关键概念，并把术语翻译成直觉
5. 总结论文最值钱的洞见
6. 给出类似博导交流风格的审稿判断与实用启发
7. 基于内置模板生成最终的 Org-mode 笔记

## 输出约定

- 将 Denote 风格的 Org 笔记写入 `~/Documents/notes/`
- 使用 skill 内置的 Org 文件头和命名规则
- 写入完成后汇报最终文件路径

## 主要支撑资源

- `references/template.org`：最终输出结构模板
- `SKILL.md` 内置的 Org-mode 与 Denote 命名规范
- 针对语气、清晰度和实用性的验收规则

## 关键约束

- 图表必须保持 ASCII-only
- 输出必须遵循 Org-mode 语法，而不是 Markdown 习惯
- 解释优先使用白话，不鼓励学术术语堆叠
- 遇到看不懂或论文没说清的部分，应明确说明，不能猜测

## 说明

- 该技能强调“理解并带走能用的东西”，不是正式 peer review。
- 它更适合想提炼思想、迁移方法或快速消化论文的人。
