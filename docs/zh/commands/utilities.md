## enhance-prompt

系统化增强用户提示，通过会话记忆上下文和意图分析，将模糊请求转换为可执行的技术规范。

### 使用方法

```bash
/enhance-prompt "修复登录按钮"
/enhance-prompt "重构支付模块"
/enhance-prompt "添加用户认证"
```

### 增强管道

`意图翻译` → `上下文集成` → `结构化输出`

### 意图翻译规则

| 用户表述 | 翻译为 | 关注点 |
|----------|--------|--------|
| "fix" | 调试并解决 | 根因分析，保留现有行为 |
| "improve" | 增强/优化 | 性能/可读性 |
| "add" | 实现功能 | 集成 + 边界情况 |
| "refactor" | 重构质量 | 保持行为不变 |
| "update" | 现代化 | 版本兼容性 |

### 上下文集成策略

增强时会综合以下上下文来源：

- **会话记忆**：对话历史、之前的分析结论
- **隐含技术需求**：从讨论中推断的技术约束
- **用户意图模式**：识别用户的真实目标

### 输出结构

```
INTENT:     [明确的技术目标]
CONTEXT:    [会话记忆 + 代码库模式]
ACTION:     [具体实施步骤]
ATTENTION:  [关键约束]
```

### 示例

```bash
# 输入: "修复登录按钮"
INTENT:     调试不可用的登录按钮
CONTEXT:    会话中讨论过 OAuth 流程，已知状态问题
ACTION:     检查事件绑定 → 验证状态更新 → 测试认证流程
ATTENTION:  保留现有 OAuth 集成
```

### 触发场景

- 模糊语言："fix"、"improve"、"clean up"
- 需要澄清的含糊请求
- 复杂技术需求或架构变更
- 关键系统：认证、支付、安全

## version

显示本地和全局安装版本，从 GitHub 获取最新发布版本，并在有新版本时提供升级建议。

### 使用方法

```bash
/version
```

### 执行流程

1. **本地版本检查** - 读取 `./.claude/version.json`
2. **全局版本检查** - 读取 `~/.claude/version.json`
3. **远程版本获取** - 通过 GitHub API 获取最新稳定版和最新提交
4. **比较与建议** - 对比版本并提供升级建议

### 输出场景

**已是最新版本：**
```
当前已是最新稳定版 (3.2.1)
```

**有可用更新：**
```
有新的稳定版本可用: v3.2.2
当前版本: 3.2.1

升级方式:
PowerShell: iex (iwr -useb https://raw.githubusercontent.com/.../install-remote.ps1)
Bash: bash <(curl -fsSL https://raw.githubusercontent.com/.../install-remote.sh)
```

**开发版本：**
```
当前运行的是开发版本 (3.4.0-dev)
高于最新稳定版 (v3.3.0)
```

### 重要说明

- 需要网络访问以通过 GitHub API 检查远程更新
- 网络请求使用 30 秒超时
- 版本比较使用 `sort -V` 进行语义化版本对比
- 不依赖 `jq`，使用 `grep`/`sed` 解析 JSON
