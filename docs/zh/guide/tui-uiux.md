# TUI 界面修复与 UI/UX 优化说明

本文档记录 `install_tui.py` 启动的 Textual TUI（平台选择页为主）的可见性修复、视觉重构与交互体验优化方案。所有改动均为主题与样式层的增强，不改变任何既有业务逻辑与交互流程。

## 1. 问题诊断与修复

### 1.1 现象

- 平台选择页中平台列表看起来“空白/不可见”，仅有一条灰色区域；
- 顶部标题栏与底部提示栏不按预期占满屏幕，内容集中在屏幕中间。

### 1.2 根因

- 平台选择页对 `Screen` 级别设置了 `align: center middle;`，导致 Dock（顶部/底部栏）与主区域的布局预期被破坏，整体内容被居中压缩；
- 样式中使用 `$text-muted` 等语义色，但主题未显式提供对应变量，在部分渲染环境下会导致文本颜色对比不足，从而“看起来像没渲染”。

### 1.3 修复策略

- 恢复平台选择页为全屏纵向布局：由主区域负责居中内容，`Screen` 本身不再整体居中；
- 在主题层补齐 `text-muted` / `text-subtle` 等语义色，确保文本对比度稳定；
- 为列表容器提供合理的最小/最大尺寸与滚动策略，避免在不同终端尺寸下被压缩到不可读。

## 2. UI/UX 重构优化

### 2.1 视觉层次

- 采用 “Deep Space” 深色背景 + 紫色主品牌色 + 青色辅助色的组合；
- 使用 `surface/panel` 构建卡片层级（背景 → 卡片 → 分割/边框），减少大面积纯黑导致的阅读疲劳；
- 标题、提示、正文分层：标题强调（bold + primary），次级信息使用 muted 色。

### 2.2 交互反馈

- 列表高亮态采用“主色浅底 + 左侧指示条”，更符合现代列表选择控件的识别习惯；
- 按钮增加可见的键盘焦点边框（focus），提升键盘可用性；
- 输入框保留聚焦边框高亮，降低误操作成本。

## 3. 响应式与可访问性（终端环境）

### 3.1 响应式

TUI 的“跨设备/跨浏览器”更接近于“跨终端窗口尺寸/字体/主题”的兼容：

- 容器宽度采用百分比自适应，并限制最小/最大宽度；
- 列表高度使用 `min/max-height` 的自适应区间，必要时出现滚动条；
- 布局在窄窗口下优先保证内容可读与焦点可见。

### 3.2 可访问性

- 键盘导航：保留默认 Tab/方向键交互，强化 focus 可见性；
- 对比度：通过主题语义色保障文本对比度，避免“低对比导致不可见”的问题。

## 4. 性能优化

- 将平台选择页的内嵌 CSS 移除，统一交由 `styles.tcss` 管理，减少重复与冲突；
- 样式规则以“少而关键”为原则，避免过多控件级覆盖导致维护成本与渲染差异。

## 5. 验证与交付标准

### 5.1 自动化验证

- 已通过项目现有测试集（`pytest`）全量验证，确保改动不影响现有功能逻辑。

### 5.2 手动验证建议（本地）

- 启动 TUI：运行 `python install_tui.py`；
- 检查平台列表项是否可读、上下移动高亮是否明显；
- 在不同终端窗口尺寸下（窄/宽、低/高）观察布局是否仍保持可用；
- 使用 Tab 在列表、输入框、按钮之间切换，确认焦点样式明显。
