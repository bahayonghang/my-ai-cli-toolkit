# 文档类型规范

## README.md
适用于项目根目录，快速让人理解项目。
**结构**:
- 一句话描述项目是什么。
- 特性
- 快速开始 (安装, 基本用法)
- 文档链接
- 贡献指南链接
- 许可证
**要点**: 30 秒内让读者理解项目价值，提供可复制粘贴的代码示例。

## API 文档
适用于库、SDK、服务的接口文档。
**结构**:
- Overview
- Authentication
- Endpoints / Methods (Parameters, Returns, Example, Errors)

## 架构文档
适用于解释系统设计和决策。
**结构**:
- Overview
- System Diagram (Mermaid)
- Components (职责, 技术栈, 依赖)
- Data Flow
- Design Decisions (背景, 决策, 理由, 后果)

## 用户指南
适用于最终用户的操作说明。
**结构**:
- Getting Started (Prerequisites, Installation, First Steps)
- Features (What it does, How to use, Tips)
- Troubleshooting (症状, 原因, 解决)
- FAQ

## 写作风格
- **直接**: 用主动语态，"点击按钮" 而非 "按钮应该被点击"
- **简洁**: 删除不必要的词
- **专业**: 使用正确的术语
- 代码示例：直接复制运行，包含导入，展示输出
