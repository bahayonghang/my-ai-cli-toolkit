# Site Patterns

这个目录用于沉淀特定网站的已验证经验。目录可以为空；为空时表示当前没有可复用的站点经验，不影响 `web-access` 正常使用。

## 何时读取

- 已经知道目标域名，怀疑仓库中可能存在对应经验
- 某个平台多次出现相同的 URL 结构、反爬模式、登录要求或页面陷阱

读取前可先运行：

```bash
bash "<skill-dir>/scripts/match-site.sh" "用户请求或目标域名"
```

如果脚本返回匹配结果，再读取对应文件；如果没有返回，直接按通用模式执行即可。

## 何时新增

只有在你**实际完成过该站点任务**，并且确认了稳定可复用的模式后，才新增或更新站点经验。

适合记录的内容：

- 稳定的 URL 结构或导航入口
- 已验证有效的操作策略
- 已确认存在的登录要求、懒加载特征或反爬现象
- 已反复踩中的页面陷阱

不要记录：

- 未确认的猜测
- 一次性偶发现象
- 依赖个人账号、临时会话或不可复现上下文的内容

## 文件命名

- 每个站点一个文件，文件名使用主域名，例如 `xiaohongshu.com.md`
- 如果存在多个别名，在 frontmatter 的 `aliases` 中补充

## 文件模板

```markdown
---
domain: example.com
aliases: [示例, Example]
updated: 2026-03-23
---
## 平台特征
这里写已经验证过的架构、反爬行为、登录要求、内容加载方式等事实。

## 有效模式
这里写已验证有效的 URL 模式、操作策略、入口页面或关键选择器。

## 已知陷阱
这里写什么会失败、为什么失败，以及应该如何绕开。
```

## 维护要求

- 更新 `updated` 日期
- 只写经过验证的事实
- 如果旧经验失效，优先修正文案，不保留误导性描述
