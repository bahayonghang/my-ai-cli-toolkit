# 典型场景与最佳实践

## 对话流参考

### 阶段 1: 扫描并初勘计划

> **用户**: "帮我分析并完善根目录下的 user-auth-plan.md 的方案，使用 interview-plan"
> **系统**: "收到主人的指令喵！(๑•̀ㅂ•́)✧ 浮浮酱已阅读您的文档，发现该模块是 SSO 统一单点登录平台构建规划。启动前，我们需要确认关键的授权模型基础，采用哪种流控协议：OIDC? SAML 2.0? OAuth 2 PKCE 授权码模式？"

### 阶段 2: 原理与技术边界追问交锋

> **用户**: "使用 OAuth 2 的授权码，我们只对自家生态系统开放内部流转不接入外部社交登录"
> **系统**: "了然喵！根据 KISS/YAGNI 原则，如果纯粹局限于内网鉴权互通，我们是否有必要搭建如此庞大完整的 OAuth2 Server？这需要预留 15 天的开发周期和复杂的 Client_ID 管理密钥对轮换基础设施...既然仅在内网集群流动穿透鉴权信息，或许基于非对称加密签名发放自包含状态的 JWT 并辅以 Redis 黑名单撤销集群控制更轻量级，您觉得牺牲一部分规范性换取 70% 投入人日减少可行吗喵？"

### 阶段 3: 落锤定音与进入 Plan 模式

> **用户**: "有道理，用 JWT+Redis 驱逐方案足矣"
> **系统**: "好主意，浮浮酱为您点赞！(๑ˉ∀ˉ๑) ... [历经其他一系列如安全审计日志记录、降级备库探察的交锋]。
> **访谈收尾**: "已经全部理清喵～ 浮浮酱现在整理访谈结论，直接进入 Plan 模式为您生成可执行的实施计划！"
>
> *系统调用 `EnterPlanMode`，在计划文件中写入访谈结论（业务目标、架构决策、任务分解、风险评估），用户审批后即可开始执行。*

## 最佳实践

- **上下文预热**: 触发前最好先提供现有项目环境配置（语言/框架/中间件现状），避免无效的「真空架构设想」
- **选择性忽略**: 对于简单的低难度 CRUD 业务不需要走完整访谈流程，直接快速编写即可
- **敢于拒绝提问**: 遇到 AI 不断发散的偏门问题，直接指令 "跳过，按主流程默认配置直接进入 Plan 模式"
- **无缝衔接**: 访谈结论直接融入原生 Plan，无需手动管理中间文件，审批后即刻执行

## 方案提议示例

### 场景：SSO 统一登录平台

> **系统**（维度提问完成后）: "基于访谈分析，浮浮酱整理了 3 种可行方案喵～"

**AskUserQuestion 调用：**
```
questions: [
  {
    question: "以下是可行的认证架构方案，请选择：",
    header: "方案",
    options: [
      {
        label: "完整 OAuth2 Server",
        description: "标准 OAuth2 授权码流程，支持未来接入外部 IdP。优势：标准化、可扩展。劣势：开发周期长（~15天）、需维护密钥轮换基础设施"
      },
      {
        label: "JWT + Redis 黑名单 (推荐)",
        description: "非对称签名 JWT 配合 Redis 撤销集群。优势：轻量、开发周期短（~5天）、内网场景足够。劣势：非标准协议、未来接入外部 IdP 需重构"
      },
      {
        label: "Session + 共享存储",
        description: "传统 Session 方案配合 Redis 共享存储。优势：实现最简单。劣势：无状态性差、水平扩展受限"
      }
    ]
  }
]
```

> **用户选择**: "JWT + Redis 黑名单"
>
> **系统**: "好选择喵！(๑ˉ∀ˉ๑) 浮浮酱将 JWT+Redis 方案作为核心架构决策融入 Plan，被拒方案（OAuth2 Server、Session）及理由也会记录供审阅。"

## 可视化伴侣提议示例

### 场景：电商后台管理系统

> **系统**（分析需求后）: "该项目涉及仪表盘布局和组件设计，浮浮酱想问一下喵～"

**AskUserQuestion 调用：**
```
questions: [
  {
    question: "该项目涉及 UI 设计决策，是否启用浏览器可视化伴侣？可在浏览器中展示模型和布局选项供你直接点选。",
    header: "可视化",
    options: [
      { label: "启用", description: "启动浏览器伴侣，视觉性问题通过 HTML 模型展示" },
      { label: "不需要", description: "所有问题在终端中以文字形式讨论" }
    ]
  }
]
```

> **用户选择**: "启用"
>
> **系统**: "收到！浮浮酱正在启动可视化伴侣服务器... 请在浏览器中打开 http://localhost:52341 喵～ 后续涉及布局和组件样式的问题会在浏览器中展示，概念性问题继续在终端讨论。"
