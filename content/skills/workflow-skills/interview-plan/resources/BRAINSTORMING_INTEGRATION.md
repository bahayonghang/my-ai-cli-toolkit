# 头脑风暴集成协议

本文档定义访谈过程中三个可选的头脑风暴增强环节。所有环节均可通过用户说"跳过头脑风暴"一次性跳过。

## 1. 范围分解检查（步骤 2.5）

### 触发条件

分析需求文档后，如果检测到以下任一情况则触发：
- 涉及 3 个或以上独立子系统/模块
- 需求文档超过 500 行
- 存在明显可独立交付的功能块

### 协议

使用 `AskUserQuestion` 向用户确认：

```
questions: [
  {
    question: "该项目涉及多个独立子系统，建议拆分为独立计划逐个推进。是否拆分？",
    header: "范围",
    options: [
      { label: "拆分推进", description: "将各子系统拆为独立计划，逐个完成访谈和实施" },
      { label: "整体推进", description: "保持为单一计划，一次性完成所有子系统的访谈" },
      { label: "先聚焦核心", description: "仅对核心子系统进行访谈，其余后续再议" }
    ]
  }
]
```

如果用户选择拆分，则为每个子系统分别执行后续访谈流程。

## 2. 可视化伴侣提议（步骤 3.5）

### 触发条件

仅在以下情况提议：
- 项目涉及 UI/前端开发
- 需求包含布局、组件设计、视觉风格等视觉性决策
- 用户明确提到界面设计相关需求

### 协议

使用 `AskUserQuestion` 提议：

```
questions: [
  {
    question: "该项目涉及 UI 设计决策，是否启用浏览器可视化伴侣？可在浏览器中展示模型和布局选项供你直接点选。",
    header: "可视化",
    options: [
      { label: "启用", description: "启动浏览器伴侣，视觉性问题通过 HTML 模型展示" },
      { label: "不需要", description: "所有问题在终端中以文字形式讨论" }
    ]
  }
]
```

如果用户选择启用：
1. 读取 `resources/VISUAL_COMPANION.md` 获取完整操作指南
2. 运行 `scripts/start-server.sh --project-dir <项目路径>` 启动服务器
3. 后续涉及视觉性决策的问题，按 VISUAL_COMPANION.md 中的交互循环执行
4. 访谈结束后运行 `scripts/stop-server.sh $SCREEN_DIR` 清理

### 逐问题决策

启用后并非所有问题都用浏览器。每个问题独立判断：
- **视觉性问题**（布局对比、组件样式）→ 浏览器
- **概念性问题**（技术选型、架构决策）→ 终端

## 3. 方案提议协议（步骤 4.5）

### 触发时机

完成 INTERVIEW_DIMENSIONS.md 的 A-G 维度提问后、进入风险保护检查前。

### 协议

基于访谈中收集的信息，提出 2-3 种实现方案，每种方案包含：
- 方案名称（简洁标签）
- 核心思路（1-2 句话）
- 权衡分析（优势与劣势）
- 推荐理由（如果有推荐）

使用 `AskUserQuestion` 呈现方案选择：

```
questions: [
  {
    question: "基于访谈分析，以下是可行的实现方案，请选择：",
    header: "方案",
    options: [
      {
        label: "方案 A: <名称>",
        description: "<核心思路>。优势：<...>。劣势：<...>"
      },
      {
        label: "方案 B: <名称>",
        description: "<核心思路>。优势：<...>。劣势：<...>"
      },
      {
        label: "方案 C: <名称> (推荐)",
        description: "<核心思路>。优势：<...>。劣势：<...>"
      }
    ]
  }
]
```

### 方案融入结论

用户选定方案后，在步骤 6（整理访谈结论）中：
- 将选定方案作为"架构决策"的核心内容
- 简要记录被拒方案及拒绝理由（供 Plan 审阅者理解决策背景）
- 在步骤 7 的 Plan 中体现选定方案的具体实施路径

### 方案数量指南

- **简单项目**（单一技术栈、明确路径）：可提出 2 种方案
- **复杂项目**（多技术栈、架构级决策）：提出 3 种方案
- **极简项目**（仅一种合理路径）：跳过方案提议，直接说明推荐路径并确认

## 跳过机制

用户在访谈任意阶段说出以下关键词，将跳过所有头脑风暴增强环节：
- `跳过头脑风暴`
- `skip brainstorming`
- `直接进入计划`

跳过后，访谈流程回退到原始步骤序列（步骤 1-8），不执行范围分解、可视化伴侣和方案提议。
