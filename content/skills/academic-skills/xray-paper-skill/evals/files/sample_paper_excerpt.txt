Title: Sparse Memory Routing for Industrial Anomaly Detection
Authors: Lin Qiao, Mira Patel
Venue: Workshop on Reliable Industrial AI 2025

Abstract
We study anomaly detection on industrial logs where events are long, repetitive,
and sparsely informative. Existing transformers over-attend to frequent but
uninformative tokens, which hurts rare fault detection. We propose Sparse Memory
Routing (SMR), a two-stage model that first writes compressed event summaries
into a memory bank and then routes only a subset of memory slots into the fault
classifier. On two internal manufacturing datasets, SMR improves F1 by 4.2 points
over the strongest transformer baseline while reducing inference cost by 37%.

Method
The key idea is that most log tokens repeat normal operating patterns, so the
classifier should not consume every token equally. SMR first encodes windows of
logs into memory slots. A routing module scores which slots are likely to contain
fault-relevant context, and only those slots are passed to the classification head.
We supervise the router weakly using fault timestamps when they are available.

Limitations
We assume timestamps are approximately aligned with true fault onset, and we do
not test settings where the monitoring system misses large parts of the logs.
Our experiments are limited to two factories from the same vendor ecosystem.
