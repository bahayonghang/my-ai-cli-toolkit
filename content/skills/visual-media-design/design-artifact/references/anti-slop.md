# Anti-Slop Guidance

Use this checklist to keep the artifact sharp across product, landing, deck, and motion work.

## Avoid generic AI artifact patterns

Do not fall back to:

- a hero plus three feature cards when the request is really about product behavior
- a fake dashboard full of random charts and numbers
- a slide deck made of title, three bullets, and "thank you"
- motion that exists only to look flashy without explaining anything
- "modern gradient" styling as a substitute for hierarchy, pacing, or interaction design

## Avoid false scope

Keep the artifact honest about what it is trying to prove:

- product prototype -> show task flow and states
- landing page -> show message arc and calls to action
- deck or storyboard -> show sequence and pacing
- motion demo -> show timing and change

Do not mix all artifact types into one noisy page unless the user explicitly wants that.

## Avoid shallow variants

If the user asks for multiple directions, vary meaningful dimensions:

- structure
- information hierarchy
- interaction model
- pacing
- visual intensity

Do not create three versions that only change color, radius, or background texture.

## Avoid decorative overload

Be careful with:

- too many badges or status chips
- icon soup
- glassmorphism everywhere
- over-styled shadows
- meaningless textures or glow layers
- transitions on every element

Decoration is only useful when it clarifies the idea or strengthens the intended mood.

## Avoid fake complexity

Do not add controls or sections just to make the artifact look more complete:

- unused filters
- dead tabs
- empty metric rows
- fake testimonials
- meaningless roadmaps
- placeholder nav items with no role in the artifact

Every visible element should earn its place.

## Preserve credibility

If something looks interactive, it should work or clearly read as static.

If a section claims to tell a story, the story should actually progress.

If the artifact borrows from existing code or screenshots, keep the density and language believable instead of replacing them with generic copy.

## Typography and spacing

Keep hierarchy obvious:

- strong title-to-body contrast
- clear spacing rhythm
- disciplined line lengths
- enough empty space for scanability
- labels that explain rather than decorate

Typography should guide reading order, not just make the artifact look "designed."
