# Artifact Modes

Choose one mode before generating the HTML artifact.

## 1. Concept Screen

Use when the user needs one strong product surface:

- homepage-like app shell
- dashboard concept
- settings redesign
- search or creation workflow entry point

Output shape:

- one primary screen
- optional secondary panels or overlays
- minimal interaction, strong hierarchy

## 2. Flow Prototype

Use when the user cares about a user journey:

- onboarding
- checkout
- workspace creation
- approval flow
- multi-step form

Output shape:

- 2-5 linked screens or states
- obvious next / back / progress logic
- each screen should push the task forward

## 3. Interactive Demo

Use when the user wants the artifact to demonstrate behavior, not just layout:

- filters changing results
- tabs switching views
- cards expanding
- a settings toggle affecting the interface
- inline validation or feedback

Output shape:

- one HTML file with simple JavaScript interactions
- enough interactivity to communicate the product idea clearly
- no unnecessary framework simulation

## 4. Comparative Exploration

Use when the user wants choices:

- “show me 2-3 directions”
- “compare onboarding approaches”
- “explore a few layouts”

Output shape:

- multiple directions inside one HTML artifact when practical
- each direction should differ in structure, task framing, or interaction strategy
- avoid trivial color-only variants

## Selection Rules

- If the user mentions a single screen or surface, start with **Concept Screen**.
- If the user mentions a journey, funnel, or sequence, use **Flow Prototype**.
- If the value is in behavior or state change, use **Interactive Demo**.
- If the user explicitly wants options, use **Comparative Exploration**.

When in doubt, prefer the mode that best reveals the product task rather than the one that looks most visually impressive.
