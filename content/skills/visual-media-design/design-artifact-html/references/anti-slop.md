# Anti-Slop Guidance

Use this checklist to keep the HTML prototype sharp.

## Avoid generic AI product patterns

Do not fall back to:

- a hero section plus three feature cards when the user asked for a product prototype
- dashboards filled with random charts that do not serve a task
- fake analytics numbers with no product meaning
- a left sidebar + top bar shell unless the product truly needs that structure
- “modern SaaS” gradients as a substitute for interaction design

## Avoid shallow variants

If the user wants multiple directions, vary meaningful dimensions:

- task structure
- information hierarchy
- interaction model
- navigation pattern
- density / pacing

Do not create three copies that only differ in color or card radius.

## Avoid decorative overload

Be careful with:

- too many badges
- too many icons
- over-styled shadows
- excessive glassmorphism
- noisy background effects

The artifact should explain the product idea first.

## Avoid fake complexity

Do not add UI elements just to make the prototype feel “complete”:

- unused filters
- dead toggles
- empty tabs
- meaningless notification bells
- fake profile menus with no role in the task

Every visible control should help explain how the product works.

## Preserve interaction credibility

If an element looks interactive, it should either work in the prototype or clearly read as static.

Prefer a smaller number of convincing interactions over a large number of broken or implied ones.

## Typography and spacing

Keep hierarchy obvious:

- strong title to body contrast
- disciplined spacing rhythm
- compact copy
- enough white space for scanability

Do not use typography merely as decoration; use it to guide task comprehension.
