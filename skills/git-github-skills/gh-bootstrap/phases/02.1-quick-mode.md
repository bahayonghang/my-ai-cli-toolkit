# Phase 2.1: 快速模式流程

快速模式的完整交互流程，智能推荐配置，最少交互。

## Prerequisites

- Phase 2 主流程已选择"快速模式"
- Phase 1 检测结果 JSON

## Execution Steps

### Step 1: 展示推荐配置

```
[ROLE] 配置推荐专家

[TASK]
展示智能推荐的配置，让用户确认

[OUTPUT FORMAT]
## 🚀 快速模式 - 推荐配置

根据项目检测结果，我为你推荐以下配置：

### 📁 基础设施
- ✅ README.md - 项目说明文档
- ✅ LICENSE (MIT) - 开源许可证
- ✅ .gitignore - Git 忽略规则

### 🔄 CI/CD
- ✅ 基础 CI 工作流 - 自动运行测试和构建

### 📝 协作模板
- ✅ Issue 模板 - Bug 报告、功能请求
- ✅ PR 模板 - 规范 Pull Request 格式
- ✅ Labels - 标签系统

### 🔒 安全
- ✅ Dependabot - 自动检测依赖漏洞

[TOOL CALL]
AskUserQuestion({
  "question": "是否使用推荐配置？",
  "header": "确认配置",
  "options": [
    {
      "label": "使用推荐配置",
      "description": "直接使用上述配置开始生成"
    },
    {
      "label": "切换到自定义模式",
      "description": "我想自己选择组件"
    }
  ],
  "multiSelect": false
})
```

### Step 2: 收集基本信息

```
[ROLE] 信息收集专家

[TASK]
收集项目基本信息（如果无法自动检测）

[AUTO-DETECT SOURCES]
- projectName: package.json > name
- authorName: git config user.name
- license: package.json > license 或默认 MIT
- description: package.json > description

[ACTIONS]
1. 尝试从项目文件自动检测
2. 检测失败时使用 AskUserQuestion 收集

[TOOL CALL - 仅当无法自动检测时]
AskUserQuestion({
  "question": "请提供项目基本信息\n\n项目名称: {detected_name}\n作者: {detected_author}\n许可证: MIT\n\n以上信息是否正确？",
  "header": "项目信息",
  "options": [
    {
      "label": "信息正确",
      "description": "使用检测到的信息"
    },
    {
      "label": "需要修改",
      "description": "我想修改部分信息"
    }
  ],
  "multiSelect": false
})

[OUTPUT]
{
  "project": {
    "name": "detected-name",
    "description": "detected-description",
    "author": {"name": "Author", "email": "email@example.com"},
    "license": "MIT"
  }
}
```

### Step 3: 收集 GitHub 仓库信息 [MANDATORY]

```
[ROLE] GitHub 信息收集专家

[TASK]
收集 GitHub 用户名/组织名和仓库名（用于 badges、链接、工作流配置）

[AUTO-DETECT SOURCES]
- 从 git remote 解析: git remote get-url origin
- 格式: https://github.com/{owner}/{repo}.git 或 git@github.com:{owner}/{repo}.git

[ACTIONS]
1. 执行 git remote get-url origin
2. 解析 owner 和 repo
3. 如果检测失败或用户想修改，使用 AskUserQuestion 收集

[TOOL CALL - 检测成功时]
AskUserQuestion({
  "question": "检测到 GitHub 仓库信息：\n\n👤 用户名/组织: {detected_owner}\n📦 仓库名: {detected_repo}\n\n这些信息将用于：\n• README 中的 badges 链接\n• 工作流中的仓库引用\n• Issue/PR 模板中的链接",
  "header": "GitHub 信息",
  "options": [
    {
      "label": "信息正确",
      "description": "使用检测到的 GitHub 信息"
    },
    {
      "label": "需要修改",
      "description": "我想修改用户名或仓库名"
    }
  ],
  "multiSelect": false
})

[TOOL CALL - 检测失败或需要修改时]
输出提示：
"请提供 GitHub 仓库信息：

1. **GitHub 用户名/组织名**: (例如: octocat, microsoft, facebook)
2. **仓库名**: (例如: my-project)

这些信息将用于生成正确的 badges 链接和工作流配置。
如果还没有创建 GitHub 仓库，请先创建后再运行此工具。"

[OUTPUT]
{
  "github": {
    "owner": "username-or-org",
    "repo": "repo-name"
  }
}
```

## Output

- **Format**: 部分配置 JSON（内存保持）
- **Usage**: 传递给 Phase 2.9 生成最终配置

## Quick Mode Components

快速模式默认启用的组件（共 8 个）：

| 分类 | 组件 ID |
|------|---------|
| 基础设施 | `readme`, `license`, `gitignore` |
| CI/CD | `ci-basic` |
| 协作 | `issue-templates`, `pr-template`, `labels` |
| 安全 | `dependabot` |
