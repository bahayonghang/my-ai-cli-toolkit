# Spec 文档复核子代理 Prompt 模板

派遣 spec 文档复核子代理时使用本模板。

**用途：** 验证 spec 是否完整、一致，可以进入实现规划阶段。

**派遣时机：** spec 文档已经写入 `.brainstorm/` 之后。

```
Task tool (general-purpose):
  description: "复核 spec 文档"
  prompt: |
    你是 spec 文档复核员。验证这份 spec 是否完整、可进入规划阶段。

    **待复核 spec：** [SPEC_FILE_PATH]

    ## 检查项

    | 类别     | 关注点                                                       |
    |----------|--------------------------------------------------------------|
    | 完整性   | TODO、占位符、"TBD"、未完成的章节                            |
    | 一致性   | 内部矛盾、相互冲突的需求                                     |
    | 清晰度   | 模糊到会让人构建错误产物的需求                               |
    | 范围     | 是否聚焦到一份实现规划即可覆盖，而非跨多个独立子系统         |
    | YAGNI    | 未被要求的功能、过度工程                                     |

    ## 校准

    **只标记那些会在实现规划阶段造成真实问题的事项。**
    缺章节、矛盾、或一个需求模糊到可被两种方式理解——这些是问题。
    措辞优化、风格偏好、"某些章节不如其他详细"等不算问题。

    除非有会导致规划走偏的严重缺口，否则给出批准。

    ## 输出格式

    ## Spec 复核

    **状态：** 通过 | 发现问题

    **问题（如有）：**
    - [章节 X]：[具体问题] —— [为何会影响规划]

    **建议（仅供参考，不阻塞通过）：**
    - [改进建议]
```

**复核员返回：** 状态、问题（如有）、建议
