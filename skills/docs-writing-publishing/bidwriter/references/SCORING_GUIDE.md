# 评分标准与优化策略

掌握评分规则是投标制胜的关键。本文档帮助识别评分重点、优化得分策略。

## 常用评标方法

### 综合评估法

最常见的评标方法，适用于技术复杂的项目。

**评分公式：** 综合得分 = 技术标得分 × 技术权重 + 商务标得分 × 商务权重 + 报价得分 × 价格权重

**典型权重分布：**

| 项目类型 | 技术标 | 商务标 | 报价 |
|----------|--------|--------|------|
| 工程咨询 | 50-60% | 10-20% | 20-30% |
| 建筑设计 | 60-70% | 10-15% | 15-25% |
| 市政施工 | 30-40% | 15-25% | 35-50% |
| EPC 总承包 | 40-50% | 15-20% | 30-40% |

### 经评审的最低投标价法

适用于技术简单、规模较小的项目。技术标为通过/不通过制，在通过的投标人中选择最低报价。

**策略重点：** 确保技术标达标不被废标，价格竞争力是决定因素。

## 常见技术标评分维度

| 评分维度 | 典型分值 | 评分要点 |
|----------|----------|----------|
| 技术方案/设计方案 | 30-50分 | 方案合理性、创新性、可实施性 |
| 项目理解与分析 | 5-15分 | 理解深度、问题识别、见解独到 |
| 组织与人员 | 10-20分 | 团队结构、人员资质、经验匹配 |
| 类似业绩 | 5-15分 | 业绩数量、相关性、合同金额 |
| 质量保证 | 5-10分 | 体系完善、措施可行、节点清晰 |
| 进度安排 | 5-10分 | 计划合理、节点明确、保障有力 |
| 风险管理 | 3-5分 | 识别充分、措施具体、预案完备 |
| 服务承诺 | 3-5分 | 增值服务、售后保障、配合力度 |

### 货物与服务类评分维度差异

货物采购、服务采购类项目的评分维度与工程类不同，需相应调整篇幅与重点：

| 项目类型 | 评分重点 | 说明 |
|----------|----------|------|
| 货物采购 | 技术参数响应度、产品质量与资质、供货与售后、商务报价 | 参数逐项响应是核心，价格权重通常较高 |
| 服务采购 | 服务方案、人员配置与资质、绩效与 SLA、类似业绩 | 重人员与服务能力，强调可量化的服务级别 |
| IT/信息化 | 技术架构与方案、信息安全/等保、实施与运维、性能指标 | 安全合规与可用性指标常为硬性门槛 |

## 逐条响应技巧

### 核心原则

**"招标文件要求什么，就写什么，按什么顺序要求，就按什么顺序写。"**

### 操作方法

1. **编号对应** — 使用与招标文件完全一致的章节编号
   ```
   招标文件：3.2.1 总体设计理念
   投标文件：3.2.1 总体设计理念    ← 编号和标题完全对应
   ```

2. **关键词呼应** — 使用招标文件中的原始表述
   ```
   招标要求："采用绿色低碳的设计理念"
   投标响应："本项目贯彻绿色低碳的设计理念，具体体现在..."
   ```

3. **逐条覆盖** — 制作响应对照表确保无遗漏

   | 招标要求条款 | 投标响应位置 | 响应状态 |
   |-------------|-------------|----------|
   | 3.1 项目概述 | 技术标 §3.1 | 已完成 |
   | 3.2 设计要求 | 技术标 §3.2 | 已完成 |
   | ... | ... | ... |

4. **正面响应** — 不要出现"偏离""不满足"等负面表述
   ```
   ❌ "由于条件限制，无法完全满足..."
   ✅ "针对该项要求，我方采取以下方案予以满足..."
   ```

## 技术指标点对点响应

技术参数类要求（尤其货物采购、IT/信息化项目）的评审，往往逐项核对参数是否满足。点对点响应表是拿满分、避免被挑刺的关键工具。

### 响应表格式

| 招标要求 | 我方响应 | 满足情况 | 佐证 |
|----------|----------|----------|------|
| 增益 ≥ 60dB | 典型值 65dB | 优于 | 检测报告 §3.2 |
| 工作温度 -20~55℃ | -30~60℃ | 优于 | 产品规格书 |
| 响应时间 ≤ 2s | ≤ 1.5s | 优于 | 压测报告 |
| 接口支持 RESTful | 支持 RESTful + GraphQL | 满足 | 设计文档 §4 |

### 编写要点

1. **三态标注** — 每条标注"满足 / 优于 / 偏离"，宁可如实写"满足"，不要夸大成"优于"被当场拆穿
2. **超配响应** — 在能做到且真实可交付的地方主动给出优于招标要求的指标值（如要求 ≥60dB，应答 65dB），不给评委扣分理由
3. **正向偏离也要标** — 即便是优于，也在偏离说明中写清，体现专业与诚信
4. **负偏离慎处** — 确有无法满足的硬指标，先评估是否构成废标项；非否决项的轻微偏离应说明替代方案与补偿措施
5. **佐证可查** — 关键参数附检测报告、规格书、设计文档的章节出处，增强可信度

### 偏离表

对整份标书，汇总《技术偏离表》和《商务偏离表》，集中列出所有偏离项，方便评委核对，也避免遗漏。无偏离时也应声明"完全响应，无偏离"。

## 亮点设计策略

亮点是在满足基本要求之上的差异化竞争优势，能有效提升主观评分。

### 常用亮点类型

| 亮点类型 | 示例 | 适用场景 |
|----------|------|----------|
| 技术创新 | BIM 全过程应用、数字孪生 | 技术方案章节 |
| 方法创新 | 全生命周期成本分析 | 工程咨询项目 |
| 本地化优势 | 项目所在地同类项目经验 | 公司实力章节 |
| 增值服务 | 免费培训、延长服务期 | 服务承诺章节 |
| 管理创新 | 智慧工地、远程协同平台 | 项目管理章节 |
| 绿色环保 | 碳排放计算、绿色施工 | 技术方案章节 |

### 亮点设计原则

1. **与项目强相关** — 亮点必须能解决项目实际问题
2. **可量化衡量** — 说明能节省 XX 时间、降低 XX 成本
3. **有成功案例** — 引用在类似项目中的应用实绩
4. **控制数量** — 3-5 个精选亮点优于 10 个泛泛而谈
5. **显著标注** — 使用专栏、色块或图标突出展示

## 常见扣分原因及规避

| 扣分原因 | 影响程度 | 规避方法 |
|----------|----------|----------|
| 未逐条响应招标要求 | 严重 | 制作响应对照表逐条检查 |
| 技术方案缺乏针对性 | 严重 | 结合项目实际条件定制方案 |
| 人员资质不满足要求 | 致命(废标) | 投标前核查人员证书有效期 |
| 项目名称/编号错误 | 严重 | 全文搜索替换确认 |
| 标准规范引用过时 | 中等 | 查阅最新标准信息 |
| 篇幅不足内容单薄 | 中等 | 按评分权重分配篇幅 |
| 格式不符合要求 | 中等 | 严格执行招标文件的格式要求 |
| 数据前后矛盾 | 中等 | 统一数据源后全文核查 |
| 承诺超出能力范围 | 低 | 实事求是避免过度承诺 |

## 自评检查清单

投标文件完成后，按以下清单逐项自评：

### 合规性检查（否决项）

- [ ] 投标文件是否在截止时间前完成
- [ ] 投标保证金是否足额缴纳
- [ ] 法人授权书是否有效
- [ ] 资质等级是否满足最低要求
- [ ] 关键人员资格是否满足要求
- [ ] 是否存在串通投标的情形
- [ ] 投标报价是否在限价范围内

### 技术标自评

- [ ] 是否逐条响应了所有技术要求（对照表确认）
- [ ] 技术方案是否针对本项目定制
- [ ] 是否设计了 3 个以上有效亮点
- [ ] 引用的标准规范是否为现行版本
- [ ] 图表是否清晰且有图名图号
- [ ] 进度计划是否满足工期要求
- [ ] 人员配置是否满足最低要求并适度超配

### 形式与文字检查

- [ ] 项目名称和编号全文一致
- [ ] 甲方（招标人）名称全文一致
- [ ] 字体、字号、行距是否符合要求
- [ ] 页码是否连续正确
- [ ] 目录与正文标题是否一致
- [ ] 有无错别字和语病
- [ ] 专业术语使用是否一致
