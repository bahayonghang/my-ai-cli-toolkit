# 阅读协议：概览前的四步预检

完整深读在进入逐图解读之前，先按顺序完成：

**叙事类型判断 → 阅读前 3 件事预检 → 作者思考路径重建 → 整体概览 → 逐图解读**

---

## 1. 叙事类型判断（概览前必做）

进入整体概览之前，先判断这篇文章的核心叙事类型，因为两类论文的阅读重心完全不同。

**判断方式**：读 Abstract + Introduction 最后一段

| 叙事类型 | 特征 | 阅读重心 |
|---|---|---|
| **方法/工具型** | "我们提出了 X 方法/架构/系统/框架" / 有 benchmark 表格 / 以 SOTA 或效率提升为卖点 | 看 ablation 设计是否合理；baseline 选择是否公平；提升是否在真实、大规模或分布外设置下仍成立 |
| **科学发现型** | "我们发现了 X 现象/规律/机制"（如缩放律、涌现能力、训练动力学、表征结构）/ 以证据链为核心 | 看证据是否支持结论；有无替代解释；结论的推广条件（模型规模、数据分布、任务族）是否清晰 |

**输出格式**（放在概览之前）：

```
📐 叙事类型：[方法/工具型 | 科学发现型]
核心卖点：[一句话]
阅读策略：[本次解读重点关注什么]
```

如果发表场景与叙事类型不一致（如一篇本质是工程系统的论文投到偏理论的会场，或反之），明确指出并说明用哪套策略。

---

## 2. 阅读前 3 件事预检（概览前必做）

在正式概览之前，先确认三件事。它们是后续逐图解读、批判分析和 follow-up 生成的主线。

1. **核心 claim**：论文最想让读者相信的技术 claim 或科学 claim 是什么？
2. **关键证据链**：支撑这个 claim 的关键实验、关键图表、关键 benchmark 或关键论证是什么？
3. **关键 baseline / prior work**：这篇论文最应该和谁比较？是某个方法、某个系统、某个数据集/benchmark、某个理论结果，还是同方向的一组代表性工作？

如果这三件事暂时不清楚，先不要急着讲方法细节。回到 Abstract、Introduction 末段、实验小节标题、图表标题和 Discussion 开头寻找线索。

### 2.1 Novelty 校准（必要时执行）

如果创新性不清楚，先搜索或调用文献工具查看 2-3 篇最相关论文，用来校准这篇论文真正新在哪里。

**校准重点**：

- 是新问题、新数据/benchmark、新设定、新方法、新证据链，还是旧方法在新场景的应用？
- prior work 已经能做到什么？这篇论文具体推进了哪一步？
- 作者声称的 novelty 是否被实验或证据链支撑？
- 如果只是新数据集或新任务包装，要说明它为什么仍然有价值，或者为什么贡献有限。

不要凭感觉宣布它 novel，也不要为了生成 follow-up idea 编造相关工作。如果只做了有限检索，要明确说明校准范围有限。

---

## 3. 作者思考路径重建（重要，概览前必做）

这一步要在正式讲方法之前完成。目标是**逆向工程作者的 idea 来源**，而不是复述论文说了什么。

**核心约束**：不能把论文自己的贡献当作前提。只能使用这篇论文发表之前已经存在的背景知识——先前方法的失败模式、经验观察、理论线索、相邻领域的工具、领域内公认的未解问题。

**重建逻辑**：

1. **起点**：这个方向在这篇论文出现之前，做到了什么？哪些方法/架构是标配？已知的瓶颈在哪（算力、数据、样本效率、泛化、延迟、可扩展性）？
2. **矛盾点**：有没有某个现象或失败案例，让人觉得"现有方法不够用"？（Introduction 和 Related Work 里通常有线索）
3. **跨越点**：从已有知识到这篇论文的 idea，中间那一步"灵感跳跃"是什么？作者可能受到了哪个相邻领域或类比的启发（如把控制论引入训练、把检索引入生成、把编译器思路引入调度）？
4. **为什么是现在**：这个 idea 为什么偏偏出现在这个时间点？是算力到了、某类数据可得了、某个基础模型/工具链成熟了，还是某个新理论框架刚出现？

**输出要求**：

- 用连贯段落描述，不要用 bullet 列表堆砌。
- 明确区分"有文本证据支持的推断"和"合理的思维实验"。
- 如果这篇论文的 idea 来源比较直接（如工程改进型论文），可以简短说明并跳过深度重建。

---

## 4. 整体概览（必须先完成）

提供结构化概览，包括：

- **一句话总结**：这篇文章解决了什么问题，用了什么方法，得到什么结论。
- **研究背景**：为什么要做这个研究，领域内的知识空白或工程痛点。
- **核心创新点**：方法上或发现上的新颖之处；如已执行 novelty 校准，要说明它相对 prior work 新在哪里。
- **规模与设置**：模型规模/参数量、数据规模、算力预算、benchmark 覆盖、评测设置等关键数字。
- **主要技术路线**：简要的方法流程或系统架构。
- **设计轨迹**：作者为什么做出这些设计选择？（如：为什么用这种注意力变体而非标准注意力？为什么选这个学习率调度？为什么这样切分数据？重建关键选择背后的驱动逻辑，而不只是总结它说了什么。）
