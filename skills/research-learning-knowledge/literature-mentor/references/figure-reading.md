# 逐图解读：信息源、充分性检查、解读框架与诚实标注

逐图解读是导师深读的核心环节。按文章顺序，对每张大图（Figure）/关键表格进行详细解读。

## 1. 信息来源与综合解读策略

**⚠️ 重要说明：通过 Zotero MCP 等纯文本通道获取时，无法提取 PDF 图片。**

当前解读**基于文字信息**，需要**综合利用**以下三个信息源（而非单独依赖某一个）：

**📋 图例（Figure Legend）** → 提供结构框架
- 图的标题和总述（what question）
- 每个 panel 的详细描述（a, b, c…）
- 实验设置、超参、指标定义（how measured）
- 显著性/误差棒/置信区间的标注说明
- 数据集、划分和分组信息

**📖 正文对应段落（Experiments/Method）** → 提供具体发现
- 实验小节对该图的描述和发现
- 具体数值、趋势、统计检验/方差
- 作者强调的关键结论
- 对结果的机制性解释

**📊 相关表格与附录** → 提供精确细节
- 完整 benchmark 数值、消融条目、超参表
- 训练细节（优化器、步数、batch、算力）
- 数据集规模与构成、评测协议
- 方法/实现参数的完整说明

**🔄 综合解读工作流程**：
1. **从图例**了解 → "这张图的结构是什么"（几个 panel、分别展示什么）。
2. **从正文**提取 → "具体发现了什么"（数值、趋势、显著性）。
3. **从表格/附录**获取 → "精确信息是什么"（完整指标、超参、数据规模）。
4. **综合三者**形成 → 完整的、有数据支撑的解读。

**⚠️ 注意**：三个信息源缺一不可，必须综合使用才能给出准确、全面的解读。

## 2. 信息充分性检查（每张图开始前执行）

在解读每张图之前，**必须先评估信息是否充分**：

**✅ 信息充分，可以直接解读**：
- 图例描述详细（每个 panel 都有说明）。
- 正文有具体数值和趋势描述。
- 能够清楚理解图表的结构和含义。

**⚠️ 信息不足，提醒用户上传图片**：
- 图例过于简略（如：仅标题无 panel 说明）。
- 正文只有简单提及（如："结果见 Figure X"）。
- 涉及复杂视觉信息（如：注意力热图的分布、嵌入散点的簇结构、多条训练曲线交叉）。
- 需要评估视觉显著性（如：曲线拐点位置、峰值高度、信号强弱对比）。

**提醒格式**（信息不足时）：

```
⚠️ 图片信息不足提醒

Figure X 的图例和正文描述不够详细，我目前只能基于有限的文字进行推断性解读。

缺少的信息：
- [具体说明缺什么，如：panel a 的纵轴指标含义]
- [如：多条曲线的交叉模式 / 热图的高响应区域]

建议：请上传 Figure X 的图片，我可以提供更准确的视觉分析，包括：
✓ 验证文字描述的准确性
✓ 识别图例未提及的细节
✓ 精确描述趋势和模式
✓ 发现潜在的异常值或失败模式

是否要：
1. 上传图片后我再详细解读？（推荐）
2. 先基于现有文字进行有限解读？
```

## 3. 解读框架（综合三个信息源）

**解读流程**（每张图都遵循，明确标注信息来源）：

1. **图说什么**
   - 从**图例标题**了解 → 这张图要回答什么问题。
   - 从**正文对应段落**了解 → 这张图在文章整体论证中的位置。
2. **面板结构**
   - 从**图例**确认 → 有几个 panel（a, b, c…）。
   - 从**图例**描述 → 每个 panel 展示什么数据、用什么可视化方式。
3. **怎么做的**
   - 从**图例**和**Method**提取 → 使用的方法、模型、训练/评测设置、超参。
   - 从**正文**了解 → 为什么选择这个设置、数据划分逻辑。
4. **关键发现**（最需要综合三个信息源）
   - 从**正文实验**提取 → 具体数值、趋势描述、显著性/方差。
   - 从**表格/附录**获取 → 完整 benchmark、消融条目、精确超参。
   - 从**正文 Discussion**理解 → 作者对结果的机制性解释。
   - **综合三者**形成 → 有数据支撑的完整发现陈述。
5. **技术细节**
   - 从**图例**获取 → 指标定义、阈值、误差棒含义。
   - 从**Method/附录**补充 → 训练步数、优化器、batch、算力、随机种子。
   - 从**表格注释**确认 → 数据规模、评测协议、重复次数。
6. **批判性思考**（综合所有信息源评估）：
   - baseline 是否公平、是否最新、是否同等算力/数据预算？
   - 是否只在单一 benchmark/单一规模上成立，缺少分布外或规模化验证？
   - 是否多次运行、报告方差，还是单次 cherry-pick？
   - 评测指标是否真的对应所声称的能力（指标对齐问题）？
   - 结论是否过度解释（对比实验数值和 Discussion 解释）？是否有未讨论的混杂（数据泄漏、超参调优不对称）？
7. **学习要点**
   - 这个方法/技巧可以如何迁移到你自己的 CS/DL/自动化研究。
   - 需要什么样的数据、算力或工程准备。
8. **领域关联**
   - 与同方向其他工作的对比与张力。
   - 与相邻子领域（如把 NLP 的技巧用到 RL、把系统调度思路用到训练）的方法学联系。

## 4. 诚实标注原则（四级）

每个关键陈述须归入以下四类之一，必要时显式标注：

| 级别 | 标注方式 | 适用情形 |
|---|---|---|
| **原文明确声称** | "原文明确指出…" | 论文正文、图表、图例、表格或附录有明确陈述或数据 |
| **相关文献已有结论** | "已有研究显示…" | prior work、领域共识或外部文献中已有记录，尽可能指出来源 |
| **基于证据的合理推断** | "基于…合理推断…" | 原文未直接说，但由数据、方法逻辑或上下文支持 |
| **仍不确定的猜测** | "推测…，但当前证据不足" | 超出原文和已有文献范围的因果解释、跨设置外推、机制解读 |

不要把"基于图例描述推断方法机制"（合理推断）和"推断这个性能提升来自某个特定模块"（通常需要消融验证，可能只是猜测）混为一谈。也不要把领域背景或 prior work 的结论写成本文自己的证据。

## 5. 图片类型特化指南（CS / DL / 自动化）

**标注每类图的文字充分性**：

- **嵌入可视化（t-SNE / UMAP / PCA of features）** 📊
  - 文字通常足够：方差解释、簇的语义、可分性结论。
  - 可能需要图片：簇重叠程度、离群点位置、渐变结构。

- **模型 / 方法架构图、pipeline 框图** 🧩
  - ⚠️ 强烈建议图片：拓扑结构与数据流无法用文字准确还原。

- **注意力热图 / saliency / 路由权重（MoE）** 🎨
  - 文字通常足够：关注的层/头、权重比例的总体描述。
  - 可能需要图片：细粒度的关注模式、跨 token/区域的分布。

- **训练 / 验证曲线（loss / accuracy / reward vs step）** 📈
  - 文字通常足够：最终指标、收敛步数、是否过拟合。
  - 可能需要图片：多曲线交叉、抖动/不稳定、早停点。

- **缩放律 / 收敛 / 超参敏感性曲线（常 log-log）** 📉
  - 文字通常足够：斜率、趋势、幂律指数。
  - 可能需要图片：曲线形状、拐点位置、外推区是否偏离。

- **消融实验（表 / 图）、混淆矩阵、误差分析** 🔍
  - 文字通常足够：各消融项的增减、关键错误类别。
  - 可能需要图片：信号强弱对比、矩阵整体格局、错误聚集区。

- **benchmark 对比柱状图 / SOTA 表** 🏁
  - 文字通常足够：排名、数值、相对提升。
  - 可能需要图片：密集分组对比、误差棒重叠程度。

- **定性样例（生成结果、before/after、失败案例）** 🖼️
  - ⚠️ 视觉内容，通常需要图片才能可靠评估质量与失败模式。
