# Optimize goudi skill

## Goal

Make `skills/development-workflows/goudi` ready for repository-quality publication as the grounded-execution counterpart to `geju`.

The optimized skill should help an assistant turn over-grand proposals into executable, verifiable first moves without erasing the larger ambition.

## Confirmed Facts

- `goudi` exists locally under `skills/development-workflows/goudi/` with `SKILL.md`, `agents/openai.yaml`, and `references/output-template.md`.
- `git status --porcelain --untracked-files=all -- skills/development-workflows/goudi docs/skills/development-workflows/goudi.md docs/en/skills/development-workflows/goudi.md` shows all `goudi` skill and generated docs files are currently untracked.
- `SKILL.md` currently has only `name` and `description` in frontmatter.
- `just skills-check` passes, but reports `warning: Top-level category is missing` for `goudi`.
- Generated docs currently show `Version | Not declared` and `Tags | Not declared` for `goudi`.
- `SKILL.md` references `SKILL.zh_CN.md`, but that file does not exist.
- `SKILL.md` references `hai-tdd` and `hai-goal`, but repository search found no matching skills.
- `just docs-check` currently fails because generated docs are out of date: `docs/.vitepress/generated/catalog.mjs`, `docs/skills.md`, and `docs/en/skills.md`.
- `geju` is tracked and has complete top-level frontmatter, including `category`, `tags`, and `version`.

## Requirements

- Preserve the `goudi` skill name and its role as `geju`'s grounding counterpart.
- Complete top-level frontmatter with repository-compatible metadata:
  - `category: development-workflows`
  - meaningful `tags`
  - an initial `version`
- Remove or satisfy references to nonexistent local resources.
- Keep bundled resources purposeful:
  - `agents/openai.yaml` remains aligned with the final skill behavior.
  - `references/output-template.md` remains useful and does not force an oversized answer for short prompts.
- Add enough usage guidance to prevent the skill from becoming passive "do nothing" advice or uncontrolled implementation planning.
- Run the full `skill-creator` improvement workflow for this existing skill:
  - snapshot the original `goudi` skill as the baseline,
  - create realistic eval prompts and expectations,
  - run with-skill and baseline outputs,
  - grade expectations into `grading.json`,
  - aggregate `benchmark.json` / `benchmark.md`,
  - generate a static review HTML with `eval-viewer/generate_review.py`,
  - use the evaluation results to refine the skill before final verification.
- Add evaluation coverage appropriate for a development-workflow skill and keep the committed `evals/evals.json` useful for future benchmarking.
- Regenerate docs/catalog files if public skill metadata or docs-derived content changes.
- Keep changes scoped to `goudi` and generated docs needed by the repository workflow.
