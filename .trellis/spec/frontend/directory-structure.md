# Directory Structure

> How the docs-site frontend is organized in this repository.

## Overview

`docs/` is a standalone VitePress site. Authored pages stay in Markdown, generated catalog output stays generated, and the site configuration stays in `.vitepress/`. Do not invent an app-style `src/` tree for this repo.

## Directory Layout

```text
docs/
├── index.md
├── en/
│   └── index.md
├── skills.md
├── en/
│   └── skills.md
├── hooks.md
├── en/
│   └── hooks.md
├── commands.md
├── en/
│   └── commands.md
├── skills/
│   └── <category>/<skill>.md
├── en/
│   └── skills/<category>/<skill>.md
├── .vitepress/
│   ├── config.mts
│   └── generated/catalog.mjs
└── scripts/
    ├── ensure_docs_deps.py
    └── sync_docs_catalog.py
```

## Module Organization

- `docs/index.md` and `docs/en/index.md` are authored landing pages.
- `docs/skills.md`, `docs/hooks.md`, and `docs/commands.md` are generated index pages.
- `docs/skills/<category>/<skill>.md` and `docs/en/skills/<category>/<skill>.md` are generated detail pages.
- `docs/.vitepress/config.mts` owns site metadata, nav, and sidebar wiring.
- `docs/.vitepress/generated/catalog.mjs` is generated by `docs/scripts/sync_docs_catalog.py` and should not be hand-edited.
- `docs/scripts/` owns docs-only maintenance and generation helpers.

## Naming Conventions

- Keep path symmetry between Chinese and English pages.
- Generated skill page names mirror the skill slug from `skills/<category>/<skill-name>/SKILL.md`.
- Keep all docs filenames lowercase and kebab-case when they are authored or generated by the catalog script.
- Keep site config filenames exact: `config.mts`, `catalog.mjs`, `sync_docs_catalog.py`.
